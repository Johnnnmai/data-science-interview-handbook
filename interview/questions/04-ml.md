# Machine Learning Concepts

Based on candidate reports from Reddit, Glassdoor, and personal blogs about what they were actually asked in data science interviews.

This section covers ML interview questions for DS roles. These are conceptual and applied questions - not coding implementations. Interviewers test whether you understand the trade-offs behind model choices, not whether you can recite sklearn API calls.


## Format

Typically 45-60 minutes, conversational. The interviewer asks conceptual questions and follows up with "why" and "when" questions. Some companies pair this with a whiteboard exercise where you draw out a model pipeline or describe a training workflow.

ML questions in DS interviews differ from MLE interviews in an important way: data scientists are expected to explain when and why to use a method, not how to implement it from scratch. The focus is on intuition, trade-offs, and connecting model choices to business outcomes.


## Interview Questions

### Supervised vs. Unsupervised Learning

- Explain the difference between supervised and unsupervised learning. Give a concrete example of each from a product analytics context [^glassdoor-google-ds]
- When would you use clustering (unsupervised) instead of classification (supervised) for user segmentation? What are the trade-offs? [^glassdoor-meta-ds]
- You have a dataset of user behavior but no labels. The PM wants to identify "at-risk" users. How would you approach this problem? [^glassdoor-netflix-ds]
- What is semi-supervised learning? When would a data scientist use it in practice? Give a realistic scenario [^interviewquery-ml]
- Explain the difference between classification and regression. You are predicting customer lifetime value. Is this a classification or regression problem? Could it be framed as both? [^glassdoor-airbnb-ds]
- What is self-supervised learning and why has it become important? How does it relate to foundation models? [^reddit-ds-ml]


### Bias-Variance Tradeoff

- Explain the bias-variance tradeoff in your own words. Use a concrete example [^glassdoor-google-ds]
- You train a decision tree and it achieves 99% accuracy on the training set but 65% on the test set. What happened? How do you fix it? [^glassdoor-meta-ds]
- You train a logistic regression and it achieves 60% accuracy on both training and test sets. What happened? How do you fix it? [^interviewquery-ml]
- How does model complexity relate to bias and variance? Draw the relationship [^glassdoor-netflix-ds]
- What is the difference between underfitting and overfitting? For each, name two techniques to address it [^glassdoor-uber-ds]
- You are choosing between a linear model with 70% accuracy and a neural network with 85% accuracy. What factors beyond accuracy should influence your decision? [^reddit-ds-ml]
- Explain how cross-validation helps you detect overfitting. Why is it better than a single train-test split? [^glassdoor-airbnb-ds]


### Regularization

- What is regularization? Why do we need it? [^glassdoor-google-ds]
- Explain the difference between L1 (Lasso) and L2 (Ridge) regularization. When would you use each? What happens geometrically? [^glassdoor-meta-ds]
- You have 500 features and suspect many are irrelevant. Which regularization approach would you use and why? [^interviewquery-ml]
- What is elastic net and when does it make sense to use it over pure L1 or L2? [^glassdoor-netflix-ds]
- How do you choose the regularization strength (lambda)? What happens when lambda is too large? Too small? [^reddit-ds-ml]
- Beyond L1/L2, what other regularization techniques exist? Discuss dropout, early stopping, and data augmentation [^glassdoor-uber-ds]


### Feature Engineering

- What is feature engineering and why does it matter more than model selection for most DS problems? [^glassdoor-google-ds]
- You are building a churn prediction model. The raw data has: user signup date, last login date, number of purchases, total spend, support tickets filed. What features would you engineer? [^glassdoor-meta-ds]
- What is the difference between one-hot encoding and label encoding for categorical variables? When is each appropriate? What problems does high cardinality create? [^interviewquery-ml]
- Explain feature scaling. When is it necessary? When can you skip it? Name two scaling methods and their trade-offs [^glassdoor-airbnb-ds]
- What is feature selection? Compare filter methods (correlation, mutual information), wrapper methods (recursive elimination), and embedded methods (L1 regularization). When would you use each? [^glassdoor-netflix-ds]
- You have a datetime column. What features can you extract from it? Which would be useful for predicting daily sales at a restaurant? [^reddit-ds-ml]
- What is target encoding for categorical features? What is the risk and how do you mitigate it? [^glassdoor-uber-ds]
- What is feature leakage? Give an example of how it can happen in a real DS project and how it would affect your model evaluation [^glassdoor-google-ds]


### Model Evaluation

- Explain accuracy, precision, recall, and F1 score. For a fraud detection system, which would you optimize and why? [^glassdoor-meta-ds]
- What is the ROC curve? What is AUC? Why is AUC useful for comparing models? [^glassdoor-google-ds]
- Your binary classifier has AUC = 0.92 but precision at your operating threshold is 0.3. Is this model useful? How would you decide? [^interviewquery-ml]
- Explain the precision-recall tradeoff. For a medical screening test, how would you set the classification threshold? [^glassdoor-netflix-ds]
- What is the difference between micro-averaging and macro-averaging for multi-class metrics? When does it matter? [^glassdoor-airbnb-ds]
- You are comparing two models: Model A has higher accuracy, Model B has higher recall. How do you decide which is better? [^reddit-ds-ml]
- What is a calibration curve? Why does calibration matter for a model that predicts probability of conversion? [^glassdoor-uber-ds]
- Explain k-fold cross-validation. When would you use stratified k-fold? When would you use time-series cross-validation? [^glassdoor-google-ds]
- What is the difference between offline evaluation and online evaluation (A/B testing the model)? Why might a model that looks good offline perform poorly online? [^glassdoor-meta-ds]
- Your model has great offline metrics but the PM says "it does not feel right in production." How do you investigate? [^glassdoor-netflix-ds]


### Tree-Based Methods

- Explain how a decision tree makes splits. What criteria does it use (Gini, entropy, information gain)? [^glassdoor-google-ds]
- What are the advantages and disadvantages of decision trees compared to linear models? [^interviewquery-ml]
- Explain random forest. Why does it work better than a single decision tree? What is bagging? [^glassdoor-meta-ds]
- Explain gradient boosting. How does it differ from random forest? What is the key idea behind boosting? [^glassdoor-netflix-ds]
- Compare XGBoost, LightGBM, and CatBoost. When would you choose each? [^glassdoor-uber-ds]
- What is the difference between bagging and boosting? Which is more prone to overfitting and why? [^reddit-ds-ml]
- How do you tune hyperparameters for a gradient boosting model? Which parameters matter most? [^glassdoor-airbnb-ds]
- Feature importance from a random forest shows Feature X as the most important. Can you trust this? What are the pitfalls of tree-based feature importance? [^glassdoor-google-ds]


### Ensemble Methods

- What is an ensemble method? Why do ensembles generally outperform individual models? [^glassdoor-google-ds]
- Explain stacking. How does it differ from bagging and boosting? [^interviewquery-ml]
- You have three models: logistic regression (AUC 0.78), random forest (AUC 0.82), and gradient boosting (AUC 0.84). Would ensembling these three models help? When does ensembling not help? [^glassdoor-meta-ds]
- What is a voting classifier? Explain hard voting vs. soft voting [^glassdoor-netflix-ds]
- When would you choose a simple model (logistic regression) over a complex ensemble? Give specific scenarios [^reddit-ds-ml]


### Deep Learning Basics

Note: deep learning is not the focus of most DS interviews. These questions appear when the role involves NLP, computer vision, or recommendation systems.

- Explain what a neural network is at a high level. What is a hidden layer? What is an activation function? [^glassdoor-google-ds]
- What is the vanishing gradient problem? How do modern architectures address it? [^interviewquery-ml]
- Explain the difference between CNNs and RNNs. When would you use each? [^glassdoor-meta-ds]
- What are embeddings? How are word embeddings (Word2Vec, GloVe) useful for DS tasks? [^glassdoor-netflix-ds]
- What is transfer learning? Give an example of how a data scientist (not an ML engineer) might use a pre-trained model [^glassdoor-airbnb-ds]
- Explain the transformer architecture at a high level. Why has it become dominant? [^reddit-ds-ml]
- You are asked to build a text classification model. Would you train from scratch or fine-tune a pre-trained model? What factors inform this decision? [^glassdoor-uber-ds]


### Recommendation Systems

- Explain collaborative filtering vs. content-based filtering. What are the trade-offs of each? [^glassdoor-netflix-ds]
- What is the cold start problem in recommendation systems? How would you handle it for (a) new users, (b) new items? [^glassdoor-spotify-ds]
- How would you evaluate a recommendation system? What metrics would you use beyond click-through rate? [^glassdoor-meta-ds]
- Explain matrix factorization for recommendations. What are the latent factors? [^interviewquery-ml]
- How do you handle the popularity bias problem - where your recommender keeps recommending popular items? [^glassdoor-netflix-ds]
- What is the exploration vs. exploitation tradeoff in recommendations? How do multi-armed bandits address it? [^glassdoor-uber-ds]
- You built a movie recommendation system. It recommends movies that users rate highly, but users complain the recommendations are "boring." What might be wrong? [^glassdoor-spotify-ds]


### Practical ML Decisions

- You need to build a model but only have 500 labeled examples. What approach would you take? [^glassdoor-google-ds]
- When would you use a simple heuristic or rule-based system instead of ML? Give specific examples [^glassdoor-meta-ds]
- Your model achieves 95% accuracy on a balanced dataset. On production data (1% positive rate), accuracy is still 95% but precision is 5%. What happened? [^glassdoor-netflix-ds]
- How do you handle class imbalance? Compare oversampling, undersampling, SMOTE, class weights, and threshold adjustment. When would you use each? [^interviewquery-ml]
- Explain the difference between batch prediction and real-time prediction. What factors determine which you should use? [^glassdoor-uber-ds]
- Your model needs to be interpretable because it affects loan decisions. What models would you consider and why? What is SHAP and how does it help? [^glassdoor-airbnb-ds]
- How do you monitor a model in production? What signals indicate that a model needs retraining? [^reddit-ds-ml]
- You built a churn prediction model with AUC 0.85. The PM asks "what should we do with these predictions?" How do you translate model output into actionable business decisions? [^glassdoor-meta-ds]


## How to Prepare

Focus on trade-offs over definitions. Interviewers care more about when and why you would choose a method than whether you can define it.

- Know the top 5 algorithms cold: logistic regression, decision trees, random forest, gradient boosting, k-means. For each, know strengths, weaknesses, assumptions, and hyperparameters
- For each method, practice explaining it in 2 minutes to a non-technical PM
- Practice the "why this over that" conversation: "Why gradient boosting over random forest here?" "Why not use deep learning?"
- Know evaluation metrics deeply. Precision/recall/F1 for classification, RMSE/MAE/R-squared for regression, AUC for ranking
- Understand feature engineering as the highest-leverage activity. Good features often matter more than model choice
- For recommendation system questions, know collaborative filtering, content-based filtering, and the cold start problem

Common mistakes:

- Describing how an algorithm works without explaining when to use it
- Not connecting model choices to business outcomes. "I chose this model because it optimizes revenue" is better than "I chose this model because it has higher AUC"
- Ignoring practical constraints: latency requirements, interpretability needs, data availability, team expertise
- Treating ML as the default solution. Sometimes a SQL query, a heuristic, or a lookup table is the right answer
- Not discussing model monitoring, retraining, and deployment. This signals lack of production experience
